Underwriting Workbench

Multifamily acquisitions modeled across multiple capital structures.

This prototype focuses on a single-user workflow, baseline document intake, editable assumptions, four side-by-side scenarios, and a readable underwriting engine that reviewers can inspect without extra tooling.

Current goal

Stress-test a multifamily deal before LOI, PSA, or closing.
  • Single-user MVP
  • Multifamily only
  • AI and multi-user intentionally deferred

Implementation guideposts

OCR, AI extraction, user accounts, file persistence, and audit logs are represented as documented extension points rather than fully implemented features in this draft.

The engine currently prioritizes transparent formulas and editable assumptions over deep spreadsheet parity. Your reference model can tighten the formulas in the next pass.

Export buttons are intentionally represented as future integration hooks once the output templates are supplied.
